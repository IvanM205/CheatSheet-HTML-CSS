Semantics in HTML

In programming, Semantics refers to the meaning of a piece of code â€” for example "what purpose or role does that HTML element have" (rather than "what does it look like?".)

In HTML, for example, the h1 element is a semantic element, which gives the text it wraps around the role (or meaning) of "a top level heading on your page."

Semantic Tags

Tag Name Description
<article> Article Represents a self-contained piece of content that can be independently distributed or reused.
<aside> Aside Defines content that is tangentially related to the content around it, such as sidebars or call-out boxes.
<figure> Figure Specifies self-contained content, often with a caption, and is typically used for images, illustrations, diagrams, etc.
<figcaption> Figcaption Provides a caption or legend for a <figure> element.
<footer> Footer Represents the footer for its nearest sectioning content or sectioning root element, often containing metadata about its section.
<header> Header Represents introductory content, typically a group of introductory or navigational aids.
<main> Main Represents the dominant content of the of a document.
<nav> Navigation Defines a set of navigation links.
<section> Section Represents a standalone section of content, which typically includes a heading.
